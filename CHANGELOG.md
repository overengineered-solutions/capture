v0.1.2 — 2026-06-02 — fix (codex audit): (1) `CaptureBubble` now reconciles the `active` tab when the visible tab set changes (host toggles `enabledTabs`, or `aiTab` arrives/leaves after an async role/flag resolve) — previously `active` could point at a hidden tab, leaving the strip with no active highlight and the body rendering a now-hidden panel (or, for the AI tab, nothing). (2) the modal focus trap selector now excludes non-focusable controls (`input[type=hidden]`, `[disabled]`) and filters out elements with no layout box, so the Tab-wrap `.focus()` can never silently no-op on a hidden input (the bug/feature panel emits 5+ hidden inputs).
v0.1.1 — 2026-06-02 — fix: `createOesFeedbackClient`/`createFileReportAction` now build the endpoint and validate config LAZILY (at call time), not at factory-call time. Constructing the sink adapter at module load with unset `OES_BASE_URL`/`OES_DASHBOARD_SECRET` (as `next build` does when collecting pages) no longer crashes the build with "Cannot read properties of undefined (reading 'replace')"; a missing config now fails loudly only when a report is actually filed.
v0.1.0 — 2026-06-01 — initial publish. Adapter-injected `CaptureBubble` shell (modal + popover surfaces, makeros focus-trap a11y, primopicks role-gating contract, oelm mobile/safe-area), the `./core` central-OES feedback sink client + distiller interface, `./migrations` canonical local-mirror SQL, and the `./theme` design-token contract.
