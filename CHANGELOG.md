v0.2.0 — 2026-06-02 — major: **self-styling CaptureBubble**. (1) SELF-STYLING — the bubble no longer relies on host Tailwind (v0.1.x baked utility classes into dist JS, but no consumer scans `node_modules` in its Tailwind `content`/`@source`, so those classes were never generated → the bubble fell to the top-left, unstyled, and looked different per app). The component now injects one scoped `<style id="oescap-styles">` block at runtime (idempotent, SSR-safe) and paints itself entirely with `.oescap-*` selectors — it renders pixel-correct with ZERO host CSS/config. (2) CSS-VAR THEMING (SSOT) — every visual knob is a `--oescap-*` custom property with a baked-in OES default (light-blue `--oescap-accent` #38bdf8, plus `--oescap-accent-contrast/-surface/-surface-raised/-ink/-ink-muted/-line/-radius/-z/-shadow`); the defaults ARE the OES look, an app overrides a var in its own CSS to retheme. Changing the portfolio default = edit the defaults here → version bump → propagates. The `./theme` subpath now exports the var contract (`CAPTURE_THEME_VARS`, `CAPTURE_STYLES_CSS`, `ensureCaptureStyles`); legacy `CAPTURE_DESIGN_TOKENS`/`captureThemeCss` retained (deprecated) for back-compat. (3) FAB ICON — the 💬/📝 emoji FAB is now an inline chat-bubble SVG (currentColor, accent-tinted); light-blue accent + chat bubble is the default identity. (4) ERROR-BOUNDARY SAFE — self-styling + SSR/no-document guards make the bubble safe to mount in Next's `error.tsx` / `global-error.tsx`; nothing in the client component throws if app CSS/providers/layout are absent; sinks stay injected adapters (server actions still work from error boundaries). (5) AUTO-DIAGNOSTICS (dependency-free) — a module-level capped ring buffer (last 15) hooks `window.onerror` + `'error'`/`'unhandledrejection'` (+ opt-in `console.error`); bug/feature reports auto-stamp `captured_params.diagnostics = { browser: {userAgent, language, viewport, screen, devicePixelRatio, online, referrer, capturedAt}, recentErrors: [{message, source, line, col, stack(truncated), ts}] }` (feature reports get `recentErrors: []`). Stacks/messages truncated + the serialized blob byte-clamped (≤32 KiB) to stay under the OES route's 64 KiB context ceiling. New `./diagnostics` subpath. (6) OPT-IN MASKED SCREENSHOT — an "Attach screenshot" button in the Bug AND Feature panels (OFF by default via `enableScreenshot`); on click it dynamically `import()`s **modern-screenshot** (added to `dependencies`; dynamic import keeps it out of the main chunk) and rasterizes `document.documentElement` to a PNG after toggling a `.oescap-redacting` class on `<html>` that masks form-field values (never password fields) and any `[data-capture-redact]` element. The Blob is appended to the report FormData as `screenshot` (a File); the host's `onFileReport` uploads it (the package stays storage-agnostic). Capture failures are non-fatal (inline "screenshot failed", report still sends). New `./screenshot` subpath. Kept the v0.1.2 fixes (active-tab reconcile, focus-trap selector) intact.

v0.1.2 — 2026-06-02 — fix (codex audit): (1) `CaptureBubble` now reconciles the `active` tab when the visible tab set changes (host toggles `enabledTabs`, or `aiTab` arrives/leaves after an async role/flag resolve) — previously `active` could point at a hidden tab, leaving the strip with no active highlight and the body rendering a now-hidden panel (or, for the AI tab, nothing). (2) the modal focus trap selector now excludes non-focusable controls (`input[type=hidden]`, `[disabled]`) and filters out elements with no layout box, so the Tab-wrap `.focus()` can never silently no-op on a hidden input (the bug/feature panel emits 5+ hidden inputs).
v0.1.1 — 2026-06-02 — fix: `createOesFeedbackClient`/`createFileReportAction` now build the endpoint and validate config LAZILY (at call time), not at factory-call time. Constructing the sink adapter at module load with unset `OES_BASE_URL`/`OES_DASHBOARD_SECRET` (as `next build` does when collecting pages) no longer crashes the build with "Cannot read properties of undefined (reading 'replace')"; a missing config now fails loudly only when a report is actually filed.
